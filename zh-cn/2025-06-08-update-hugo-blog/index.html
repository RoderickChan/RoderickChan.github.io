很久没有更新博客的主题了。目前采用的FixIt主题功能比较全面，就是配置稍微有些复杂。早些时候，我就在滴答清单里面写下了要更新博客主题的task，忙了一阵又一阵，昨天晚上到今天终于把这个事情给忙活完了。总的来说，工作量上除了继承主题已有的更新，例如：合集、侧边栏、相关内容推荐、右侧目录等，还实现了如下几个功能：clustrmaps显示访客地点和数量、AI全文摘要、主页添加自己的微信公众号、添加留言板等。

FixIt主题更新

这次更新属于是把配置文件重新写了一遍，并且按照作者的doc的配置风格，对自己的博客站点做了适配。大部分时间，就是在理解配置的含义，以及修改对应配置为自己博客站点所需要的字段。主要涉及的更新有以下内容。更新完后，每篇文章大概长这样：

添加合集功能

然后，在对应的文章下显示是这样的：

侧边栏

上面说的合集是放置到侧边栏的，实际上，侧边栏还有两个小组件，一个是相关内容推荐。看了下，好像是hugo根据关键字、标签、类别等文章的元数据字段计算出两篇文章的相似性，然后按照相似性的高低排序进行推荐。总的来说，丰富了博客的内容，还是相当不错的。

除了相关内容外，FixIt的主题文档左侧还添加了一个图片，作者给的是他的知识星球的链接，我就改成了微信公众号的链接。反正就是替换下图片的URL而已，没什么太大的改动量。侧边栏显示如下：

AI全文摘要

众所周知，现在是AI盛行的时代。其实主题里面作者已经实现了一个postSummary组件。但是，这个组件是基于洪墨AI - 网站智能客服和智慧搜索搭建工具这个实现的。一来要收费，二来收费还挺贵。咱也不知道他用的是什么模型，但是从网上的帖子来看，几百万token用的很快，没几篇文章总结完就又得再充钱了。不得不说，要是有时间，我也想做个类似的产品出来。毕竟实现起来非常简单。

其实自己想要实现这一功能的话，也不是很难。如果做成服务的话，涉及的防盗、错误处理等比较复杂。所以，思路如下：

  1. 调用大模型为每篇文章生成一个摘要，大约100字
  2. 把摘要内容写到博客的summary元数据上
  3. 在postContentBefore这里注入代码，写一个机器人的框框，然后使用typeit把博客的summary内容渲染出来

只需要三步，就能实现相同的功能。而且可以在本地选择模型，比如deepseek就很便宜，我的140篇文章，大约28字的博客内容，总结了140篇摘要只花了0.8人民币。如果把摘要的字缩短一点，可能耗费的token更少。

import os
import json
from pickle import TRUE
import requests
from pathlib import Path
import frontmatter
from typing import Dict, List, Optional
import threading
from concurrent.futures import ThreadPoolExecutor, as_completed
import time

class BlogSummarizer:
    def __init__(self, directory_configs: List[Dict], openai_api_key: str, model: str, api_base_url: str, output_path: str="summary.json", max_workers: int=3):
        """
        初始化博客摘要生成器
        Args:
            directory_configs: 目录配置列表，每个配置包含 {'path': str, 'prompt': str}
            openai_api_key: OpenAI API密钥
            model: 使用的模型名称
            api_base_url: API基础URL
            output_path: 摘要文件输出路径
            max_workers: 最大并发线程数
        """
        self.output_path: str = output_path
        self.directory_configs = directory_configs
        self.api_key = openai_api_key
        self.base_url = api_base_url
        self.headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        self.model = model
        self.max_workers = max_workers
        # 线程锁，用于保护共享资源
        self.lock = threading.Lock()
        # 加载已有的摘要数据
        self.existing_summaries = self.load_existing_summaries()
        
    def load_existing_summaries(self) -> Dict[str, str]:
        """
        从输出文件中加载已有的摘要数据
        
        Returns:
            以 (filename, title) 为键，摘要为值的字典
        """
        summaries = {}
        if os.path.exists(self.output_path):
            try:
                with open(self.output_path, 'r', encoding='utf-8') as f:
                    data = json.load(f)
                    for item in data:
                        key = f"{item['filename']}_{item['title']}"
                        summaries[key] = item['summary']
                print(f"✓ 加载了 {len(summaries)} 个已有摘要")
            except Exception as e:
                print(f"加载已有摘要时出错: {e}")
        return summaries
        
    def get_md_files_with_prompts(self) -> List[Dict]:
        """获取所有目录下的.md文件，并关联对应的prompt"""
        all_files_with_prompts = []
        for config in self.directory_configs:
            directory = Path(config['path'])
            prompt = config['prompt']
            if directory.exists():
                files = list(directory.rglob("*.md"))
                for file in files:
                    all_files_with_prompts.append({
                        'file_path': file,
                        'prompt': prompt
                    })
                print(f"在目录 {directory} 中找到 {len(files)} 个markdown文件")
            else:
                print(f"警告: 目录不存在: {directory}")
        return all_files_with_prompts
    
    def parse_markdown_file(self, file_path: Path, prompt: str) -> Optional[Dict]:
        """
        解析markdown文件的frontmatter
        
        Args:
            file_path: 文件路径
            prompt: 对应的摘要生成提示词
            
        Returns:
            包含文件信息的字典，如果是草稿则返回None
        """
        try:
            with open(file_path, 'r', encoding='utf-8') as f:
                post = frontmatter.load(f)
            
            # 检查是否为草稿
            if post.metadata.get('draft', True): # 没有draft字段也视为草稿
                print(f"跳过草稿文件: {file_path.name}")
                return None
            
            # 检查是否已有summary字段且存在内容
            # existing_summary = post.metadata.get('summary', '').strip()
            # if existing_summary:
            #     print(f"跳过已有摘要的文件: {file_path.name}")
            #     return None
            
            # 获取标题
            title = post.metadata.get('title', file_path.stem)
            
            return {
                'title': title,
                'filename': file_path.name,
                'filepath': str(file_path),
                'content': post.content,
                'prompt': prompt,
                'post_object': post  # 保存frontmatter对象用于后续更新
            }
            
        except Exception as e:
            print(f"解析文件 {file_path} 时出错: {e}")
            return None
    
    def get_summary_from_cache_or_api(self, file_info: Dict) -> Optional[str]:
        """
        先从缓存中查找摘要，如果没有则调用API生成
        
        Args:
            file_info: 文件信息字典
            
        Returns:
            摘要文本
        """
        # 构建缓存键
        cache_key = f"{file_info['filename']}_{file_info['title']}"
        
        # 先检查缓存（使用线程锁保护）
        with self.lock:
            if cache_key in self.existing_summaries:
                print(f"✓ 使用已有摘要: {file_info['title']}")
                return self.existing_summaries[cache_key]
        
        # 缓存中没有，调用API生成
        print(f"正在为 '{file_info['title']}' 生成新摘要...")
        return self.get_summary_from_openai(file_info['content'], file_info['title'], file_info['prompt'])
    
    def get_summary_from_openai(self, content: str, title: str, prompt: str) -> Optional[str]:
        """
        使用OpenAI API生成文章摘要
        
        Args:
            content: 文章内容
            title: 文章标题
            prompt: 摘要生成提示词
            
        Returns:
            生成的摘要文本
        """
        try:
            # 构建完整的prompt
            full_prompt = f"{prompt}\n\n标题: {title}\n\n内容: {content}"
            
            data = {
                "model": f"{self.model}",
                "messages": [
                    {
                        "role": "user",
                        "content": full_prompt
                    }
                ]
            }
            
            response = requests.post(
                self.base_url,
                headers=self.headers,
                json=data,
                timeout=30
            )
            
            if response.status_code == 200:
                result = response.json()
                summary = result['choices'][0]['message']['content'].strip()
                return summary
            else:
                print(f"API请求失败: {response.status_code}, {response.text}")
                return None
                
        except Exception as e:
            print(f"调用API时出错: {e}")
            return None
    
    def update_markdown_file(self, file_info: Dict, summary: str) -> bool:
        """
        将摘要更新到markdown文件的YAML元数据中
        
        Args:
            file_info: 文件信息字典
            summary: 摘要文本
            
        Returns:
            是否更新成功
        """
        try:
            post = file_info['post_object']
            post.metadata['summary'] = summary
            
            # 写回文件
            with open(file_info['filepath'], 'w', encoding='utf-8') as f:
                f.write(frontmatter.dumps(post))
            
            print(f"✓ 已更新文件: {file_info['filename']}")
            return True
            
        except Exception as e:
            print(f"更新文件 {file_info['filepath']} 时出错: {e}")
            return False
    
    def process_single_file(self, file_with_prompt: Dict) -> Optional[Dict]:
        """
        处理单个markdown文件
        
        Args:
            file_with_prompt: 包含文件路径和prompt的字典
            
        Returns:
            处理结果字典或None
        """
        file_path = file_with_prompt['file_path']
        prompt = file_with_prompt['prompt']
        
        try:
            # 解析文件
            file_info = self.parse_markdown_file(file_path, prompt)
            if file_info is None:
                return None
            
            # 获取摘要（从缓存或API）
            summary = self.get_summary_from_cache_or_api(file_info)
            
            if summary:
                # 更新markdown文件的YAML元数据
                if self.update_markdown_file(file_info, summary):
                    # 添加到缓存（使用线程锁保护）
                    cache_key = f"{file_info['filename']}_{file_info['title']}"
                    with self.lock:
                        self.existing_summaries[cache_key] = summary
                    
                    # 返回结果
                    return {
                        'title': file_info['title'],
                        'filename': file_info['filename'],
                        'summary': summary
                    }
            
            return None
            
        except Exception as e:
            print(f"处理文件 {file_path.name} 时出错: {e}")
            return None
    
    def process_files(self) -> List[Dict]:
        """
        使用多线程处理所有markdown文件
        
        Returns:
            包含所有文章信息和摘要的列表
        """
        files_with_prompts = self.get_md_files_with_prompts()
        results = []
        updated_files = 0
        
        print(f"\n总共找到 {len(files_with_prompts)} 个markdown文件")
        
        # 使用线程池并行处理
        with ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            # 提交所有任务
            future_to_file = {executor.submit(self.process_single_file, file_with_prompt): file_with_prompt 
                             for file_with_prompt in files_with_prompts}
            
            # 处理完成的任务
            for i, future in enumerate(as_completed(future_to_file), 1):
                file_with_prompt = future_to_file[future]
                file_path = file_with_prompt['file_path']
                
                print(f"\n处理文件 {i}/{len(files_with_prompts)}: {file_path.name}")
                
                try:
                    result = future.result()
                    if result:
                        results.append(result)
                        updated_files += 1
                        print(f"✓ 处理成功")
                    else:
                        print(f"✗ 处理失败或跳过")
                except Exception as e:
                    print(f"✗ 处理文件 {file_path.name} 时出错: {e}")
        
        print(f"\n✓ 共更新了 {updated_files} 个文件的YAML元数据")
        return results
    
    def save_summary_json(self, new_results: List[Dict]):
        """
        保存结果到JSON文件，保持已有内容并添加新结果
        
        Args:
            new_results: 新的处理结果列表
        """
        try:
            # 读取已有的JSON数据
            existing_data = []
            if os.path.exists(self.output_path):
                try:
                    with open(self.output_path, 'r', encoding='utf-8') as f:
                        existing_data = json.load(f)
                except Exception as e:
                    print(f"读取已有JSON文件时出错: {e}，将创建新文件")
            
            # 创建一个字典来快速查找已有条目
            existing_dict = {}
            for item in existing_data:
                key = f"{item['filename']}_{item['title']}"
                existing_dict[key] = item
            
            # 更新或添加新结果
            for result in new_results:
                key = f"{result['filename']}_{result['title']}"
                existing_dict[key] = result
            
            # 转换回列表并保存
            final_data = list(existing_dict.values())
            
            with open(self.output_path, 'w', encoding='utf-8') as f:
                json.dump(final_data, f, ensure_ascii=False, indent=2)
            
            print(f"✓ 摘要文件已保存到: {self.output_path}")
            print(f"✓ 总共包含 {len(final_data)} 个条目（{len(new_results)} 个新增/更新）")
            
        except Exception as e:
            print(f"保存JSON文件时出错: {e}")
